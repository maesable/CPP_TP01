1- ECHAUFFEMENT 

Ici, le compilateur affiche ; "x vaut 5"
À la ligne 6, on crée une référence r de x.
À la ligne 7, on affecte à r la valeur de y, soit 3.0.
Enfin, à la ligne 8, cette fois, r vaut 5, comme r est une référence, x vaut aussi 5.

2- PROGRAMMATION : LIVRES
 2.2- Saisie

 Dans cette liste de fonctions, les fonctions 3 et 4 sont les même pour le compilateur. Effectivement, une référence et une valeur sont la
 même chose pour lui. Donc le compilateur trouve la 4e fonction ambigue.

3- REFERENCE ET RETOUR DE FONCTION 

Ici, notre fonction swapNumberOfPages (ou swapPageCount dans le sujet), prend en argument des références alors que la fonction
createBook (ou readBook dans le sujet) renvoie un Book qui n'est pas une référence.
